What this page template should explain

Each project page should quickly show the problem, the engineering choices, and the production value. For BovEye, the story is simple: turn multispectral drone imagery into a detection system teammates can inspect, tune, and trust.

Problem

Livestock monitoring from aerial imagery needs more than a generic object detector. The system has to deal with small targets, changing altitude, vegetation, and spectral channels that normal RGB models do not expect.

Solution

Adapt the model input path for RGBN data, normalize bands consistently, and build a threshold-sweep evaluation layer so the team can choose operating points with evidence instead of guessing.

Delivery blueprint

Use this section on every project page to show how the work moved from raw data to a usable system. Keep it concrete, readable, and close to what was actually shipped.

01

Dataset and band preparation

Construct a repeatable pipeline for RGBN inputs, preserving the extra spectral channel while keeping labels, splits, and image transforms aligned.

02

Model adaptation

Modify the patch embedding path so RF-DETR can consume 4-channel imagery without throwing away pretrained structure or spectral signal.

03

Evaluation and handoff

Package results into threshold sweeps, readable summaries, and artifacts that help non-research teammates understand performance tradeoffs.
